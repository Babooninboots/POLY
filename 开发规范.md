# 开发规范.md: POLY — 交互式 LAMMPS 多晶样品生成器

作者：张杨 (Yang Zhang)

单位：Stony Brook University (纽约州立大学石溪分校)

## 1. 概述

POLY 生成用于 LAMMPS 模拟的多晶材料原子结构。它根据用户定义的参数——
盒子尺寸、晶粒尺寸分布、晶体结构、晶体学取向——构建 3D 颗粒结构，
并以标准 LAMMPS 格式输出原子模型。

PySide6 GUI 提供交互式 3D 预览、晶粒编辑和一键构建。
所有后端模块也可通过 Python 脚本无头运行。

---

## 2. 全局输入参数

| 参数               | 选项                                                                       |
| ------------------ | -------------------------------------------------------------------------- |
| **盒子尺寸**        | 起始 `(x0, y0, z0)` 和结束 `(x1, y1, z1)`，柱状/Z 轴对准模式自动调整       |
| **边界条件**        | 所有三轴周期性 (PBC)                                                        |
| **晶粒数量**        | 固定数量 或 平均晶粒直径                                                    |
| **晶粒尺寸分布**    | `random`, `normal`, `bimodal`, `customized`, `laminate`, `even`             |
| **晶体结构**        | Bravais、金属间化合物、空间群 或 自定义 (`.cif` / `.crystal`)                 |
| **取向**           | `random`, `z_alignment`, `low_angle`, `high_angle`, `custom_misorientation`, `custom_profile` |
| **输出**           | LAMMPS data 文件 (`.data`) + dump 文件 (`.dump`)                            |

## 3. 核心模块

### 模块 1：晶粒种子生成与 Voronoi 剖分 (`grain_seeds.py`)

**目标：** 确定单个晶粒的空间分布和 Voronoi 胞元边界。

1. **种子初始化**
   
   - *随机* / *正态*：均匀放置种子，然后强制执行最小间隔。
   - *双峰 (Bimodal)*：高斯混合模型，用户指定数量分数 φ、模式 1 均值/标准差
     (μ₁, σ₁) 和模式 2 均值/标准差 (μ₂, σ₂)。晶粒总数根据盒子体积和加权
     平均晶粒体积自动计算。直径从 φ·N(μ₁,σ₁) + (1−φ)·N(μ₂,σ₂) 抽取。
   - *层状*：种子在中平面上，面内随机或正态分布。
   - *均匀排列 (1D)*：种子沿一个轴等间距排列。
   - *自定义*：从 `.seed` 文件读取种子位置（3 列 `x y z` 或
     4 列 `x y z radius`）。

2. **Voronoi 剖分**
   使用 `pyvoro2` 配合 27 个 PBC 镜像（3×3×3 偏移网格）。当 `target_radii`
   可用时，运行 Laguerre-Voronoi（幂距离）剖分；否则为标准 Voronoi。

3. **尺寸分布优化 (FBSP)**
   用于 `normal` 和 `bimodal` 分布（以及 `laminate` + normal 面内）：
   力偏球堆积——一种 Monte Carlo 松弛方法，从对数正态分布（normal）或
   高斯混合分布（bimodal）抽取目标直径，并施加基于重叠的排斥力，配合
   学习率衰减。对于双峰分布，最终晶粒直径直接取自 FBSP 目标半径以保留
   双峰信号。最多运行 2000 次迭代。

4. **输出**
   返回 `SeedResult` 数据类：`seeds`、`diameters`、`target_radii`
   （用于 Laguerre-Voronoi）、`neighbors`（Voronoi 邻接关系）、
   `polyhedron_data`（每晶粒 27 个 PBC 镜像）、盒子边界。

**入口点：** `generate_grains(box_start, box_end, ...) -> SeedResult`

---

### 模块 2：完美晶体生成 (`pristine_crystal.py`)

**目标：** 构建基础晶格的单一块体超胞，中心位于原点。

1. **晶体来源**
   
   - **Bravais**（`sc`, `fcc`, `bcc`, `hcp`, `diamond`, `tetragonal`, `bct`）
     通过 `ase.build.bulk`。
   - **金属间化合物**（`L1_2`, `B2`, `D0_3`, `L2_1`, `D0_19`, `A15`, `C15`,
     `L1_0`, `rocksalt`, `zincblende` 等）通过 `ase.build.bulk`。
   - **空间群** 通过 `ase.spacegroup.crystal` 配合 Wyckoff 位置、
     空间群编号和晶胞参数。
   - **自定义** — 加载 `.cif`、`.crystal` 或任何 ASE 可读文件。

2. **超胞构建**
   复制单胞以覆盖 `coverage`（默认：2× 最大盒子对角线，
   为组装预裁剪留出余量），外加 `margin` 的额外重复。

3. **归一化**
   平移超胞使其质心位于原点。

4. **输出**
   返回 `CrystalResult` 数据类：`atoms`（ASE Atoms）、`n_atoms`、
   `repeats`（各轴重复倍数）、`unit_cell`。

**入口点：**

- `generate_pristine_bravais(element, structure, a, c, ...) -> CrystalResult`
- `generate_pristine_intermetallic(prototype, symbols, a, ...) -> CrystalResult`
- `generate_pristine_cif(path, ...) -> CrystalResult`
- `PristineCrystal` 类用于编程使用

### 模块 3：取向分配 (`orientation.py`)

**目标：** 为每个晶粒分配 ZXZ Euler 角和旋转矩阵。

1. **`random`** — 所有晶粒均匀随机 Euler 角。

2. **`z_alignment`** — 将晶体方向 `[hkl]` 对准模拟 Z 轴。面内旋转模式：
   
   - `random` — 均匀随机 φ₂
   - `low_angle` / `high_angle` — BFS 图遍历配合逐对约束检查
   - `symmetric_csl` — CSL 取向差，通过 Diophantine 方程
     `m² + N_hkl·n² = Σ` 枚举 Σ 边界

3. **`low_angle`** / **`high_angle`** — 从随机种子晶粒开始 BFS 图遍历。
   邻居被分配旋转，逐对取向差低于 10°（低）或高于 20°（高），
   通过 `min(tr(R_i·R_j^T))` 角计算。

4. **`custom_misorientation`** — Monte Carlo 模拟退火，将当前取向差分布
   与用户提供的目标分布匹配，使用 KS 统计量作为能量函数，
   按 Metropolis 准则决策。

5. **`custom_profile`** — 从 `.euler` 文件加载逐晶粒 Euler 角
   （4 列：`grain_id φ₁ Φ φ₂`）。

**输出**
返回 `OrientationResult` 数据类：`euler_angles`（N×3，度）、
`rotation_matrices`（N×3×3）、`rotations`（scipy Rotation）、
`misorientation_angles`、`mc_energy_history`、`mc_acceptance_rate`。

**入口点：** `generate_orientations(mode, n_grains, neighbors, ...) -> OrientationResult`

### 模块 4：交互式 GUI (`gui_main.py`, `gui_views.py`)

**目标：** 提供交互式 3D 预览、晶粒编辑和一键构建。

1. **布局**
   
   - **左侧停靠面板** — "全局设置"卡片，包含所有模拟输入
     （盒子尺寸、数量、分布、晶体、取向、晶粒编辑器、
     终端消息），以及暂停/停止/进度控件。
   - **左侧视图** — 3D Voronoi 多面体，Euler 角 RGB 着色，
     GPU 裁剪面（切片滑块），点选晶粒。
   - **右上视图** — 完美晶体单胞，CPK 着色原子。
   - **右下** — 晶粒尺寸直方图和取向差角直方图（pyqtgraph）。
   - **底部工具栏** — 输出文件夹、状态名称、操作按钮。

2. **操作按钮**（从左到右）
   
   - **保存完美晶体** — 写入 `{Name}.crystal`
   - **保存种子状态** — 写入 `{Name}.seed`（3 或 4 列）和 `{Name}.euler`
   - **重投** — 清除缓存，强制完全重新生成
   - **生成初始种子** — 运行种子 + Voronoi + 晶体生成
   - **开始构建** — 运行完整多晶组装

3. **交互功能**
   
   - 在 Voronoi 视图中点击晶粒选中；编辑器填充其位置和 Euler 角。
   - **应用编辑** 重新计算 Voronoi 胞元和取向差，然后重新渲染。
   - MC 优化期间 **暂停** 渲染当前中间状态。
   - **停止** 请求 MC 优化器提前终止。
   - **切片深度** 滑块提供 GPU 加速的穿过多晶体裁剪。

4. **后台工作线程** (`workers.py`)
   
   - `SeedGenerationWorker` — 用于种子 + 取向生成的 QThread。
   - `CrystalGenerationWorker` — 用于超胞构建的 QThread。
   - `PolycrystalBuildWorker` — 用于组装 + 文件输出的 QThread。
   - 工作线程发出 `progress`、`finished` 和 `error` 信号。

### 模块 5：多晶组装 (`pc_assembly.py`)

**目标：** 旋转、裁剪、平移并组装单个晶粒为最终多晶体。

1. **种子镜像网格**
   为每个晶粒种子生成 27 个 PBC 镜像（3×3×3 偏移网格）。
   在所有种子镜像上构建全局 `cKDTree`。选择每个晶粒的
   **最佳周期性镜像**——最接近模拟盒子中心的那个——
   以最小化模包裹后的 Voronoi 胞元分裂。

2. **组装流水线**
   
   **标准流水线**（random / normal / customized）：
   
   - 逐晶粒球形预裁剪，在原点进行（半径 = `grain_diag × 0.55`，
     或回退为 `0.75 × box_diag`）。
   - 按晶粒的旋转矩阵旋转预裁剪后的晶体。
   - 平移到最佳种子位置。
   - 对全局种子镜像树进行 KD-tree 查询（k=1，或当 `target_radii`
     可用时用 k=min(30,N) 进行幂距离计算）。
   - 保留最近种子为当前晶粒最佳镜像的原子。
   - **晶粒级并行化：** 当 ≥ 8 个晶粒且估计预裁剪
     ≥ 500 原子/晶粒时，通过 `ThreadPoolExecutor`（上限 8 线程）
     并发处理晶粒。每个晶粒的裁剪、旋转、KD-tree 查询和掩码
     操作均释放 GIL，产生接近线性的加速。结果按晶粒索引顺序收集，
     确保确定性输出。进度从主线程随 future 完成逐晶粒打印。
   
   **柱状流水线**（laminate / evenly spaced）：
   
   - 计算 `R_base` 将晶体 `[hkl]` 对准堆叠轴。
   - 预旋转整个晶体并裁剪为 **主柱体**（直径 =
     `2 × max(grain_diag)`，高度 = `1.5 × max_grain_z`）。
   - 逐晶粒：裁剪为半径 = `grain_diag[g]` 的圆柱，通过
     `Rz = R_total · R_base⁻¹` 面内旋转，平移到最佳种子，KD-tree 查询。
   - 堆叠轴感知的柱体尺寸，支持非 Z 堆叠方向。

3. **自适应 KD-Tree 并行化**
   当估计预裁剪超过 5,000 原子/晶粒时，`cKDTree.query`
   调用使用 `workers=-1`（全部核心）在内部并行化查询。
   低于 5,000 时，`workers=1` 避免小查询集上的线程池开销。

4. **模包裹**
   所有最终位置模包裹进入主模拟盒子：
   `positions = (positions - box_start) % box_size + box_start`。

5. **输出**
   返回 `AssemblyResult` 数据类：`positions`、`types`（1-base LAMMPS
   原子类型）、`grain_ids`（0-based）、`euler_per_atom`、`symbols`、
   `type_to_symbol`、`type_masses`、`keep_counts`（每晶粒保留原子数）。

**入口点：** `assemble_polycrystal(seeds, crystal_atoms, orientations, ...) -> AssemblyResult`

---

### 模块 6：数据输出 (`pc_assembly.py`)

**目标：** 将组装后的结构导出为 LAMMPS 可读格式。

1. **LAMMPS Data 文件** (`.data`)
   头部：原子数、类型数、盒子边界；质量段；原子段（`id type x y z`）。
   原子行通过 `numpy.savetxt` 构建到内存缓冲区，避免逐原子 Python
   `for` 循环开销（大型结构约快 50×）。

2. **LAMMPS Dump 文件** (`.dump`)
   自定义 dump 格式，用于 OVITO/ParaView 可视化：
   `ITEM: ATOMS id_POLY type x y z grain_id euler_angle_1 euler_angle_2 euler_angle_3`。
   原子行与 data 文件相同方式向量化。

3. **并行输出**
   当同时请求 `.data` 和 `.dump` 时，通过 `ThreadPoolExecutor`
   （2 线程）并发写入。进度由第一个完成的线程打印一次。

4. **Crystal 文件** (`.crystal`)
   XYZ 格式晶体保存，头部注释含晶胞矢量，可被自定义晶体来源读取。

5. **Seed 文件** (`.seed`)
   3 列（`x y z`）或 4 列（`x y z radius`）格式。4 列变体保留
   Laguerre-Voronoi 半径，以便重新加载时精确复现胞元形状。

6. **Euler 文件** (`.euler`)
   4 列：`grain_id φ₁ Φ φ₂`（ZXZ 约定，度）。
